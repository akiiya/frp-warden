# 0005 — 默认使用 SQLite(纯 Go 驱动)

- 状态:已接受
- 日期:2026-06-12

## 背景

frp-warden 应当在零外部依赖的情况下即可启动 —— 运维人员应该能够
运行单一二进制并立即拥有一个可用的数据存储。与此同时,较大规模的
部署可能希望使用联网的数据库。单一二进制、无 CGO 的目标
(参见 [0001](0001-use-go-single-binary.md))约束了驱动的
选择。

## 决策

默认使用 **SQLite**,存储于 `./data/frp-warden.db`,首次运行时自动
创建。使用**纯 Go、无 CGO 的 SQLite 驱动**(例如 `modernc.org/sqlite`),
以保持交叉编译的简单。将数据访问抽象在 `internal/db` 之后,以便
**mysql**、**mariadb** 和 **postgresql** 日后可以在不改动调用
点的前提下加入。

## 影响

- 真正的零配置启动;无需单独安装数据库。
- 交叉编译保持简单(无 CGO,无 C 版 SQLite 工具链)。
- 纯 Go 的 SQLite 驱动可能比 C 版稍慢,但对于这种控制面
  负载来说是可接受的。
- `internal/db` 抽象层与迁移脚本必须从一开始就写成可在不同驱动间
  移植,以便日后加入 mysql/mariadb/postgresql。
- 数据目录必须在初始化阶段(第 1/2 阶段)创建。
