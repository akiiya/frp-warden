# 0003 — 前端使用 Vue 3 + Vite + TypeScript(Naive UI)

- 状态:已接受
- 日期:2026-06-12

## 背景

管理控制台需要一个现代、可维护的 SPA,能够直接构建为静态资源
并嵌入到 Go 二进制中。团队倾向于使用一个对 TypeScript
支持良好、并自带成套 UI 组件的组件框架。

## 决策

使用 **Vue 3 + Vite + TypeScript** 构建前端,并以 **Naive UI** 作为
组件库。应用构建到 `web/dist`,并在第 7 阶段通过 `embed` 嵌入到 Go
二进制中。管理服务器负责提供 SPA,并对客户端路由回退到
`index.html`。

## 影响

- 快速的开发体验(Vite)与类型安全(TS)。
- Naive UI 提供了完整的组件集,减少自定义 UI 的工作量。
- 前端构建(`npm run build`)是发布二进制的前置条件。
- 前端工具链(Node.js + npm/pnpm)是开发所必需的,但运行已发布的
  二进制时并不需要。
- 如果 Naive UI 出现局限,框架选择是可以重新考量的;嵌入式
  方案与组件库的选择相互独立。
